## Features

- **Advanced Template Rendering Engine**: Generate dynamic, data-driven text through inline tags such as `{{count}}`, `{{last}}`, or `{{route=intranasal|ratio=site}}`.

- **Expressive Tag Query Language**: Execute sophisticated filtering, aggregation, analytics, and formatting through compact declarative tags.

- **Composable Query Pipelines**: Chain filters, selectors, metrics, transforms, and presentation controls within a single expression.

- **Built-in Analytical Toolkit**: Counts, ratios, rankings, frequency distributions, totals, and trend-ready outputs out of the box.

- **Dual Output Layer**: Produce polished human-readable text or structured JSON for automation, APIs, and downstream systems.

- **Schema Normalization Layer**: Resolves aliases, inconsistent field names, legacy exports, and heterogeneous datasets automatically.

- **Fault-Tolerant Processing**: Engineered to operate reliably with incomplete, noisy, or partially corrupted real-world data.

- **CLI + Python SDK**: Integrate seamlessly through terminal workflows or native Python imports.

- **Highly Configurable Core**: Centralized behavior, aliases, defaults, and rendering rules via `config.toml`.

- **Modular & Scalable Architecture**: Parser / planner / execution pipeline designed for maintainability, extensibility, and long-term growth.
